# Governance

Coverage MCP is currently maintained by Appunni M. Maintainers review changes, manage releases, respond to security
reports, and protect the project's technical direction and community standards.

Routine decisions are made through issues and pull requests using technical evidence, compatibility with the documented
project goals, and long-term maintenance cost. Significant changes to public contracts, storage, security, licensing,
or governance require a written design discussion before implementation.

Maintainer status is earned through sustained, constructive contributions and sound review judgment. Existing
maintainers add or remove maintainers publicly, except when private safety or security details must be withheld.

If the project becomes inactive, the maintainer should identify a successor or clearly mark the repository as
unmaintained rather than silently leaving users without support expectations.
